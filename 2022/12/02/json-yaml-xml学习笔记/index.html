json, yaml, xml学习笔记

Created2022-12-02|Updated2023-06-02|实用工具
|Post View:

我们在不同的计算机程序之间共享数据的时候，往往需要规范数据交换的格式。常用的格式有json，yaml和xml。

本笔记介绍了三种标记语言的语法并给出了部分示例。

  • JSON ( JavaScript Object Notation ) 是一种超轻量级的数据交换格式，因为简单易懂而且被广泛支持（已经成为ECMA标准，几乎所有编程语言都有解析JSON的库），被广泛应用于网络中交换数据。
  • YAML ( YAML Ain’t a Markup Language ) 是一个可读性高的，用来表达资料序列化的格式。YAML 是专门用来写配置文件的语言，非常简洁和强大，远比 JSON 格式方便。
  • XML ( Extensible Markup Language ) 是一种标记语言，在json出现之前大家都用xml来传递数据。XML虽然比JSON复杂，在Web中应用也不如以前多了，不过仍有很多地方在用，所以，有必要了解如何操作XML。

JSON

基本语法

JSON非常的简单，为此，其只有6种基本数据：

  • number: 可以是整数，浮点数，以及科学计数法，其本质上定义和Javascript的number定义相同。

    在JavaScript中，数字不分为整数类型和浮点型类型，所有的数字都是由 浮点型类型。JavaScript 采用 IEEE754 标准定义的 64 位浮点格式表示数字，它能表示最大值为 ±1.7976931348623157e+308，最小值为 ±5e-324。

  • boolean: 仅有两个值true和false。

  • string: 字符串可以是双引号中的任意文本。可以使用\转义字符。

  • null: 就是null。

  • array: 使用[]表示，元素之间用,分割。最后一个元素之后不应该有,。

  • object: 使用{}表示。

    元素之间用,分割，每个元素可以视为一个键值对。

一个JSON文件的内容的最外层应该是{}，这样代表一个JSON文件就是一个对象。

实例

YAML

YAML 语言（发音 /ˈjæməl/ ）的设计目标，就是方便人类读写。其实质上是一种通用的数据串行化格式。

它的基本语法规则如下。

  • 大小写敏感
  • 使用缩进表示层级关系
  • 缩进时不允许使用Tab键，只允许使用空格。
  • 缩进的空格数目不重要，只要相同层级的元素左侧对齐即可

# 表示注释，从这个字符一直到行尾，都会被解析器忽略。

YAML 支持的数据结构有三种。

  • 对象：键值对的集合，又称为映射（mapping）/ 哈希（hashes） / 字典（dictionary）
  • 数组：一组按次序排列的值，又称为序列（sequence） / 列表（list）
  • 纯量（scalars）：单个的、不可再分的值

语法

  1. XML文档必须有根元素，且整个文档应该是树形结构。

  2. XML声明（可选）。

  3. 所有的XML元素应该有一个关闭标签。

  4. XML标签对大小写敏感。

  5. XML必须是正确嵌套的。

    其形式化定义是，如果文本在某元素的开始标签和结束标签之间出现，那么称该文本出现在该元素的上下文中（in the context of），则如果每个开始标签都在同一个父元素的上下文中有唯一的结束标签与之匹配，那么该标签就是正确嵌套的。

  9. XML中，多个空格也会被保留（这和HTML行为不同）

XML数据结构

元素

XML文档的基本结构是元素 ( element ) 。一个元素就是一对互相匹配的开始和结束标签，以及之间的部分。

一个元素可以包含：

  • 其他元素
  • 文本
  • 属性
  • 以上的组合

元素的命名规则：

  • 名称可以包含字母、数字以及其他的字符
  • 名称不能以数字或者标点符号开始
  • 名称不能以字母 xml（或者 Xml、xMl 等等）开始
  • 名称不能包含空格

属性

属性是一种标记结构，在start-tag或empty-element tag内部的“名字-值对”。例如：<img src="珈百璃.jpg" alt="废天使" />。每个元素中，一个属性最多出现一次，一个属性只能有一个值。

如果属性有多个值，这需要采取XML协议以外的方式来表示，如采用逗号或分号间隔，对于CSS类或标识符的名字可用空格来分隔。
